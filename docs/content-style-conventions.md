# Content And Style Conventions

## Naming

- Use clear, literal component names.
- Prefer `TextField` over ambiguous names such as `InputShell` or `FieldThing`.
- Keep file and export names aligned for easy discovery.

## UI Copy

- Use sentence case for labels, helper text, and button text.
- Keep primary actions verb-led, for example `Save changes` or `Invite teammate`.
- Make destructive actions explicit.

## Example Code

- Keep examples realistic enough for engineers and agents to copy safely.
- Show preferred composition patterns rather than novelty layouts.
- Use examples to demonstrate variants, states, and accessibility defaults.
