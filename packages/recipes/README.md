# Fresh Recipes

This package is the prototype lane for Fresh.

Use `@fresh/recipes` when you need:

- higher-level screen blocks
- visually polished prototyping primitives
- a better PM or designer prompting surface than raw components alone
- consistent composition that engineering can still implement from easily

Use `@fresh/ui` when you need:

- shared component primitives
- lower-level reusable building blocks

Use `@fresh/shadcn-reference` when you need:

- parity guidance
- strict visual and interaction expectations

Do not import shadcn runtime code directly into Fresh apps.
