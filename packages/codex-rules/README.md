# Codex Rules

This package contains the markdown rulebooks that coding agents should follow when
generating product UI from the Fresh design system.

- `component-rules.md` explains what components to use and when.
- `screen-recipes.md` provides repeatable screen composition patterns.
- `anti-patterns.md` captures the failure modes agents should avoid.
- `taxonomy-map.md` maps shadcn catalog terms into the approved Fresh component set.
- Approved shared anatomy should be traced back to the review boards in
  `/design/board-index.md` before agents promote a new pattern into `@fresh/ui`.
